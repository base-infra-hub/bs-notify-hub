# BS-Notify-Hub

高性能多租户消息分发与通知中心基础设施。本服务作为基础架构组件，支持通过 HTTP RESTful 和高性能 gRPC 接口接收第三方系统投递的通知消息，并基于 SSE (Server-Sent Events) 长连接实时推送到多端浏览器客户端。

项目内置一个精美的暗黑玻璃拟态运维看板，提供系统连接数、消息吞吐量和租户在线分布的实时监控。部署形态为**单个二进制文件 + `config.yaml` 配置文件**。

---

## 🌟 项目亮点与核心设计

* **多协议鉴权与职责隔离**：
  * **三方接入 (JWT Only)**：面向外部服务调用的 HTTP 接口与高并发 gRPC 接口统一强制走 **RS256 JWT** 验签，校验 `exp` 过期时间与特定 `tag` 服务标志，实现零 I/O 开销的 CPU 本地快速验证。
  * **运维看板 (Session Only)**：仅限内部管理员登录使用的路由组收敛在 `/web` 空间下，使用 **Session Cookie** 进行会话管理，防止外部服务身份和管理员控制面身份发生混淆。
* **高伸缩性设计 (自适应集群/单机)**：
  * 使用 `redis.UniversalClient` 动态包装。无需改动代码，仅需在配置文件中填入单点或多个 Redis 节点，即可自适应切换 Redis 单机或集群模式。
  * 内置 `LocalBroker`（单机进程通道）和基于 Redis Pub/Sub 的 `RedisBroker`（集群广播模式），一键将服务横向扩展。
* **Ticket SSE 长连接鉴权**：
  * 浏览器原生 SSE 订阅（`EventSource`）无法携带自定义 Header（如 `Authorization`）。系统设计了 **Ticket（用后即焚临时凭证）机制**。
  * 三方系统先通过携带 JWT 的安全接口申请 Ticket，客户端携带 Ticket 建立 SSE 订阅，确保连接建立阶段的安全。
* **嵌入式前端看板**：
  * 完整的 Web 看板与登录页代码收敛在 `web/`，在 Go 编译阶段通过 `//go:embed` 机制直接打包进二进制程序中，实现真正的一键部署，无需单独维护 Nginx 与前端容器。

## 📂 项目包结构说明

项目遵循清晰、标准的 Go 项目布局分层：

```
├── main.go                     # 项目入口：加载配置、初始化服务、启动 gRPC 和 Hertz 监听
├── go.mod                      # Go 依赖描述
├── config.default.yaml         # 默认配置模板文件
├── api/                        # Protobuf 定义及生成的 gRPC 桩代码 (pb.go)
├── dist/                       # 看板前端源码与静态资源 (统一物理目录为 dist)
│   ├── index.html              # 看板主页面
│   ├── login.html              # 看板登录页面
│   └── js/                     # 轮询渲染和页面控制逻辑
├── internal/                   # 内部核心业务包 (开发逻辑)
│   ├── app/                    # 启动装配器 (初始化 DB、Redis 客户端及 Dispatcher 调度器)
│   ├── conf/                   # 配置文件解析：支持公钥格式自适应解析
│   ├── constant/               # 核心常量与 Redis Key 命名空间规则
│   ├── dispatch/               # 核心分发层：含 Broker 抽象与 Local/Redis Broker 实现
│   ├── dto/                    # 接口传输数据对象 (Request & Response)
│   ├── handler/                # HTTP 控制器 (Hub, Sender, Inbox, Status, Auth 等控制器)
│   ├── middleware/             # Hertz 中间件：JWT 校验、Session 校验、链路追踪及跨域配置
│   ├── model/                  # 数据库 GORM 数据模型定义
│   ├── repository/             # 数据访问层 (DB 关系操作与 Redis 缓存计数库)
│   ├── service/                # 核心业务逻辑层 (gRPC 业务实现、SSE 订阅、凭证申请)
│   └── tasks/                  # 后台定时任务 (过期通知定时清理)
└── pkg/                        # 通用工具包
    ├── db/                     # GORM 初始化工具
    ├── httpcode/               # 标准响应码定义
    ├── jwtutil/                # RSA 公钥解析与 JWT 验签工具
    ├── pool/                   # SSE 订阅池管理器 (线程安全)
    ├── redis/                  # UniversalClient 通用 Redis 客户端包装
    └── response/               # 统一 HTTP JSON 响应工具
```

## ⚙️ 部署指南

### 1. 环境准备
* **PostgreSQL**：建议版本 >= 12，提供通知中心的消息历史存储。
* **Redis**：单点或 Cluster 均可，用于存储订阅计数、广播集合与临时 Ticket 凭证。

### 2. 配置文件
复制 `config.default.yaml` 并命名为 `config.yaml`。
根据你所连接的数据库、Redis 以及三方鉴权服务的 RSA 验签公钥参数进行配置，将此文件与构建出的二进制程序放在同一个文件夹下。

### 3. 编译
```bash
# 编译二进制可执行程序 (去除了符号表和调试信息以缩小体积)
go build -ldflags="-s -w" -o bs-notify-hub.exe main.go
```

### 4. 运行
```bash
# 启动程序。程序会自动检测同级目录下的 config.yaml，缺失则直接报错退出
./bs-notify-hub.exe
```
服务启动后，你可以在浏览器中访问 `http://localhost:<http_port>/web` 进入运维监控看板进行监控。
